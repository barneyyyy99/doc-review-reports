## 2026-02-09

### [重大变更] 建立统一优先级标准和问题去重机制

**原因**：用户反馈三大问题：
1. 不同 review skill 对同一问题优先级判断不一致（如错别字在 .cursorrules 中是 P0，在 language-review.md 中是 P1）
2. 同一问题在不同 category 中重复出现（如"RoomMainView 导入缺失"在 code/logic/integration 中各报告一次）
3. 批量问题未合并（如代码块语言标记错误出现 10+ 次未合并）

**变更内容**：

#### 1. 新建文件
- `skills/_shared/severity-standards.md`
  - 创建统一的优先级判断标准
  - 定义 P0/P1/P2 的判断决策树
  - 补充边界案例和特殊场景说明
  - 提供不确定时的处理原则

#### 2. 修改共享层规则
- `skills/_shared/language-review.md`
  - 错别字优先级：P1 → P0（与 .cursorrules 保持一致）
  - 理由：影响专业性，降低用户信任
  - 添加统一标准引用

- `skills/_shared/code-review.md`
  - 为所有问题类型添加明确的 severity 标注（语法错误→P0、API拼写错误→P0、导入缺失→P0、代码块语言标记错误→P1等）
  - 特殊场景细化：正文与代码矛盾（核心功能→P0，注释错误→P1）
  - 添加统一标准引用

- `skills/_shared/format-review.md` - 添加统一标准引用
- `skills/_shared/logic-review.md` - 添加统一标准引用
- `skills/_shared/terms-review.md` - 添加统一标准引用

#### 3. 报告生成规则重大更新
- `skills/_shared/generate-review-report/SKILL.md`
  - **新增 Step 0.5：问题去重和合并**（强制执行）
    - 跨 category 去重：检测并合并同一问题在不同审查模块中的重复
    - 同 category 内批量合并：阈值从 ≥3 降低到 ≥2
    - 相似问题聚合：位置连续、主题相同的问题合并
    - 生成去重日志：记录合并详情便于调试
  
  - **去重规则**：
    - location 相同或重叠 + 核心问题相同 → 判定为重复
    - 保留优先级最高、描述最详细的版本
    - 在 description 中注明"该问题在X个审查模块中被发现"
  
  - **批量合并优化**：
    - 降低合并阈值：≥3 → ≥2
    - 展示 3 个典型案例
    - 提供批量修复建议（正则表达式）

#### 4. 更新顶层规则
- `.cursorrules`
  - 引用 `severity-standards.md` 作为权威标准
  - 保留快速参考，但注明详细标准见 severity-standards.md
  - 修正错别字优先级为 P0

**影响范围**：
- 所有审查模块（共享层 + 业务层）
- 所有后续生成的审查报告

**预期效果**：
1. ✅ 优先级判断完全一致
2. ✅ 同一问题不再重复报告
3. ✅ 批量问题正确合并
4. ✅ 报告质量显著提升

**注意事项**：
- 旧报告的优先级可能与新标准不一致（不影响使用）
- 建议重新审查之前标记为 P1 的错别字，升级为 P0
- 去重逻辑在报告生成时自动执行，无需手动操作
