# 通用审查标准

本文档定义了技术文档审查的通用标准，包括格式规范、文风要求和 Markdown 规范。

## 格式规范

### 文档结构
- 标题层级清晰，使用标准的 Markdown 标题（#、##、###）
- 目录结构合理，便于导航
- 章节编号或命名规范统一

### 代码块规范
- 代码块必须指定语言类型
- 代码示例完整可运行（如适用）
- 代码注释清晰，关键逻辑有说明

### 链接和引用
- 内部链接使用相对路径，确保可访问
- 外部链接有效且可访问
- API 引用准确，版本号明确

## 文风要求

### 语言表达
- 用词准确，避免歧义
- 句式简洁，逻辑清晰
- 技术术语使用一致，首次出现需解释

### 内容完整性
- 必要的前置条件说明完整
- 步骤描述清晰，无遗漏关键步骤
- 错误处理和异常情况有说明

### 受众适配
- 技术难度标注明确（入门/中级/高级）
- 背景知识要求说明清楚
- 示例场景贴近实际使用

## Markdown 规范

### 语法规范
- 标题前后空行
- 列表项格式统一（有序/无序）
- 表格格式规范，对齐清晰
- 代码块缩进正确

### 可读性
- 段落长度适中，避免大段文字
- 适当使用列表、表格、代码块等元素
- 重要信息使用强调（**粗体**、*斜体*）

### 兼容性
- 符合 CommonMark 标准
- 渲染效果在不同平台一致
- 特殊字符正确转义
