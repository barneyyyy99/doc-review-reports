# 审查问题优先级标准

> **适用范围**：本标准适用于所有技术文档审查模块（共享层和业务层）
> 
> **使用方式**：各 skill 文件中的 P0/P1/P2 标注均遵循本标准定义的规则

---

## 优先级定义

### 🔴 P0（严重）- 必须修复

**核心判断标准**：会导致用户**无法完成任务**、**理解错误**或**存在安全风险**的问题。

#### P0 问题类型清单

| 问题类型 | 典型场景 | 判断依据 |
|---------|---------|---------|
| **错别字** | "配置"写成"陪置"、"初始化"写成"出事化" | 影响专业性，降低用户信任 |
| **代码/逻辑冲突** | 正文说参数是 String，代码示例传了 Int | 导致代码无法运行 |
| **安全泄露** | 硬编码真实的 API Key、密钥、Token、内部 IP | 存在安全风险 |
| **链接失效** | 外部链接返回 404、403、500 错误 | 用户无法访问必需资源 |
| **核心信息缺失** | 集成文档未说明如何初始化 SDK | 用户无法完成核心任务 |
| **步骤遗漏/顺序错误** | 步骤 1 → 步骤 3（缺少关键步骤 2） | 导致操作失败 |
| **语法错误** | 代码缺少分号、括号不匹配、API 使用错误 | 代码无法编译或运行 |
| **导入缺失** | 使用了类但完全未导入，且前文也未导入 | 代码无法运行 |
| **前后矛盾** | 参数表说必填，示例代码中缺失 | 导致理解混乱或实现错误 |
| **示例与描述不一致** | API 文档中字段描述与示例不匹配 | 导致实现错误 |
| **关键依赖关系缺失** | 未说明功能需要先完成 X 步骤 | 用户卡在前置条件 |

#### P0 判断决策树

```
问题会导致以下任一情况？
├─ 代码无法编译/运行 → P0
├─ 用户无法完成文档承诺的核心任务 → P0
├─ 存在安全风险 → P0
├─ 理解错误导致实现完全错误 → P0
└─ 以上都不是 → 继续判断 P1
```

---

### 🟡 P1（警告）- 建议修复

**核心判断标准**：会导致用户**操作困难**、**需要额外摸索**或**理解不清晰**，但不影响最终完成任务。

#### P1 问题类型清单

| 问题类型 | 典型场景 | 判断依据 |
|---------|---------|---------|
| **格式违规** | 标题跨级（H2 → H4）、数值范围格式错误 | 影响可读性 |
| **代码块语言标记错误** | Kotlin 代码标记为 `java` | 影响语法高亮和阅读体验 |
| **表述模糊** | 使用"可能"、"应该"等不确定词汇 | 用户不确定如何操作 |
| **逻辑断链** | 要求操作但未说明原因 | 用户不理解为什么要这么做 |
| **引用缺失** | 提到"参考 XX 文档"但未提供链接 | 用户需要额外查找 |
| **正文与代码矛盾（非关键）** | 注释说"打开摄像头"但参数名是 microphone | 混淆但不影响运行 |
| **视角混乱** | "您"既指开发者又指最终用户 | 理解混乱 |
| **缺少安全提示** | Demo 配置未说明"仅用于测试" | 可能误用于生产环境 |
| **环境要求模糊** | "需要较新版本的 Xcode" | 用户不知道具体版本 |
| **验证方法缺失** | 未说明如何验证集成成功 | 用户无法确认操作正确 |
| **场景适配说明缺失** | 多种方案未说明选择依据 | 用户不知道选哪个 |

#### P1 判断决策树

```
问题会导致以下任一情况？
├─ 用户需要反复试错才能成功 → P1
├─ 用户需要额外查找资料 → P1
├─ 理解不清晰但能通过推测完成 → P1
├─ 影响阅读体验但不影响理解 → P1
└─ 以上都不是 → 继续判断 P2
```

---

### 🔵 P2（建议）- 可选优化

**核心判断标准**：纯**风格优化**，不影响功能使用和理解，用户可以正常完成任务。

#### P2 问题类型清单

| 问题类型 | 典型场景 | 判断依据 |
|---------|---------|---------|
| **中英文空格缺失** | "LiveKit提供" → "LiveKit 提供" | 纯排版风格 |
| **过度使用"您"** | 每 100 字出现 5 次以上"您" | 表达啰嗦但不影响理解 |
| **语气优化** | 被动语态 → 主动语态 | 表达风格偏好 |
| **术语不一致（轻微）** | "礼物系统"和"礼物功能"混用，但含义明确 | 不会造成混淆 |
| **缺少"常见问题"章节** | Demo 文档未包含 FAQ | 非强制要求 |
| **描述不够详细** | 可选配置项未详细展开说明 | 不影响核心功能 |

#### P2 判断决策树

```
问题属于以下情况？
├─ 纯排版/格式风格偏好 → P2
├─ 表达啰嗦但不影响理解 → P2
├─ 可选内容缺失 → P2
└─ 影响理解或操作 → 不是 P2，重新判断
```

## 特殊场景优先级判断

### 1. 代码片段不完整

**判断标准**：根据上下文和文档定位判断

| 场景 | 优先级 | 理由 |
|-----|--------|------|
| 代码声称"完整示例"但缺少必需初始化 | P0 | 用户无法运行 |
| 关键变量完全未提及来源 | P0 | 用户不知道如何获取 |
| 使用占位符+注释说明来源 | 不标记 | 开发者能理解 |
| 聚焦核心 API，省略业务逻辑 | 不标记 | 这是正常的简化 |

### 2. 错误处理缺失

| 场景 | 优先级 | 理由 |
|-----|--------|------|
| 正文明确要求处理错误但代码完全没有 | P0 | 与描述矛盾 |
| 关键操作无错误处理且会导致崩溃 | P0 | 影响稳定性 |
| 示例代码省略错误处理 | **不标记为问题** | 这是正常的简化 |
| 已有基本错误处理（即使不完善） | **不标记为问题**| 不过度要求 |

### 3. 引用缺失

| 场景 | 优先级 | 理由 |
|-----|--------|------|
| "详见 XX 文档"但未提供链接 | P1 | 增加查找成本 |
| 提到通用概念（如"Android 官方文档"） | 不标记 | 开发者能自行查找 |
| 内部文档链接格式为锚点 ID | 不标记 | 由文档平台保证 |

### 4. 不同文档类型的差异

#### API 参考文档
- **示例与描述不一致** → P0（必然导致实现错误）
- **字段描述缺失** → P0（无法正确使用 API）

#### Demo 文档
- **运行步骤缺失** → P0（无法跑通 Demo）
- **效果预览缺失** → P1（影响预期建立）

#### 集成文档
- **初始化步骤缺失** → P0（无法完成集成）
- **验证方法缺失** → P0（无法确认集成成功）

#### 功能指南
- **时序流程不完整** → P0（无法完成功能）
- **场景适配说明缺失** → P1（不知道如何选择）

## 边界案例判断指南

### 案例 1：代码块语言标记错误

**场景**：Kotlin 代码标记为 `java`

**判断**：
- ✅ **P1**（当前标准）
- ❌ 不是 P0：代码仍能复制粘贴使用
- ❌ 不是 P2：影响语法高亮，降低阅读体验

**理由**：影响开发体验但不影响功能实现

---

### 案例 2：数字与中文之间的空格

**场景**：`Android 5.0及以上` vs `Android 5.0 及以上`

**判断**：
- ✅ **不标记**（根据 2026-02-04 规则更新）
- 理由：数字与中文之间空格非强制，不影响阅读

---

### 案例 3：注释与参数名矛盾

**场景**：
```kotlin
autoEnableMicrophone = true, // 进房后是否自动开启摄像头
autoEnableCamera = true,     // 进房后是否自动开启麦克风
```

**判断**：
- ✅ **P1**（非 P0）
- 理由：参数名本身是清晰的，注释错误会造成混淆但不影响代码运行

---

### 案例 4：相同路径在不同配置文件中不一致

**场景**：
- `settings.gradle.kts` 中使用 `'room/tuiroomkit'`
- `settings.gradle` 中使用 `'live/tuiroomkit'`

**判断**：
- ✅ **P0**（导致配置错误）
- 理由：用户选择其中一个配置文件，如果路径错误会导致无法编译

## 不确定时的处理

### 原则：零误报优先

当不确定问题优先级时：

1. **标记为【需人工确认】**，不直接判定为 P0/P1
2. **在 description 中说明不确定的原因**
3. **提供判断依据**，让文档作者决定

### 示例

```json
{
  "type": "代码片段不完整【需人工确认】",
  "severity": "P0/P1",
  "description": "代码中使用了 CustomView，文档未说明其来源。如果这是项目中的自定义类，则不是问题；如果是 SDK 提供的类，则缺少导入说明。",
  "suggestion": "【请确认】如果 CustomView 是 SDK 类，请补充导入说明"
}
```

## 使用建议

### 对于审查执行者（AI Agent）

1. **首次判断**：根据本标准的决策树快速判断
2. **边界情况**：查阅"边界案例判断指南"
3. **特殊场景**：查阅"特殊场景优先级判断"
4. **仍不确定**：标记【需人工确认】

### 对于规则维护者

1. **发现新边界案例**：更新"边界案例判断指南"
2. **发现不一致**：修改本标准，同步更新各 skill
3. **记录变更**：在 `CHANGELOG.md` 中记录原因
