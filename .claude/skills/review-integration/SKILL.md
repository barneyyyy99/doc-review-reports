# 集成文档审查 Skill

## When to Use
- 审查 SDK 集成、快速接入类型的文档
- 文档目标是让用户**完成基础集成**并**快速体验核心功能**

## Instructions

**⚠️ 零误报原则（最高优先级）**
- **误报会让用户失去信任**，这比漏报更严重！
- 每个问题必须有确凿证据，不能基于推测、习惯、直觉判断
- 不确定的问题必须标记为 `【需人工确认】`

---

### Step 0：执行共享层审查（强制）
在开始任何审核前，**必须完整执行以下共享审查规则**：

1. `logic-review.md`
2. `code-review.md`
3. `format-review.md`
4. `language-review.md`
5. `terms-review.md`

> 若共享层存在 FAIL 项，需记录，但仍继续执行后续步骤。

---

### Step 1：检查文档目标说明

**判定为问题：**
- 完全缺少文档目标说明（用户不知道这篇文档是干什么的）→ **P1**
- 缺少完成后的效果说明（用户不知道完成后能达到什么状态）→ **P2**

---

### Step 2：检查前提条件完整性

#### 2.1 账号与服务开通
**检查项：**
- 是否列出需要获取的信息（SDKAppID、SecretKey、API Key 等）
- 是否说明如何获取（直接说明或提供链接）

**判定为问题：**
- 未说明需要获取哪些必要信息 → **P0**
- 未说明信息获取方式且无链接 → **P0**

#### 2.2 环境准备
**检查项：**
- 开发工具及版本（Xcode 14.0+、Android Studio 4.2+ 等）
- 目标平台版本（iOS 12.0+、Android 5.0+ 等）
- 版本号是否具体（避免"最新版本"等模糊描述）

**判定为问题：**
- 环境要求缺失或模糊 → **P1**
- 版本号使用模糊描述 → **P1**

---

### Step 3：检查集成 SDK 步骤

**检查项：**
- 是否说明安装方式（CocoaPods、Gradle、npm 等）
- 是否提供具体的安装命令或配置代码
- 是否说明在哪个文件中添加配置
- 是否提供具体的版本号

**判定为问题：**
- 未说明如何安装 SDK → **P0**
- 未提供具体的安装命令或配置代码 → **P0**
- 未说明在哪个文件中添加配置 → **P0**

---

### Step 4：检查完成集成流程

#### 核心流程完整性
**必须覆盖的步骤：**
1. 引入/导入 SDK（import 语句）
2. 初始化 SDK（位置、代码示例、配置参数）
3. 权限配置（如适用）
4. 基础 API 调用示例（⚠️ 至少一个核心 API）
5. 生命周期管理（如适用：资源释放）

**判定为问题：**
- 缺少核心流程步骤（如只说初始化，没说如何调用 API）→ **P0**
- 步骤顺序错误（如先调用 API，后初始化）→ **P0**

#### 4.1 引入/导入 SDK
- 未说明如何引入 SDK → **P0**

#### 4.2 初始化 SDK
**检查项：**
- 是否说明在哪个文件/方法中初始化
- 是否提供完整的初始化代码示例
- 是否列出必填配置项及其含义、获取方式、格式要求

**判定为问题：**
- 未说明初始化位置 → **P0**
- 未提供初始化代码示例 → **P0**
- 未列出必填配置项 → **P0**
- 配置项获取方式不明且前提条件中也未说明 → **P0**

#### 4.3 权限配置（如适用）
**检查项：**
- 是否列出必要的权限
- 是否说明在哪个文件中配置权限
- 是否提供具体的配置代码

**判定为问题：**
- 功能需要权限但完全未说明 → **P0**
- 未说明在哪个文件中配置权限 → **P0**
- 未提供具体的配置代码 → **P0**

#### 4.4 基础 API 调用示例（核心）
**检查项：**
- ⚠️ **关键要求**：必须提供至少一个核心 API 的完整调用示例
- 是否说明 API 的作用
- 是否说明必要参数的来源
- 是否说明回调/返回值的处理

**判定为问题：**
- 集成文档完全没有 API 调用示例（用户完成初始化后不知道如何使用 SDK）→ **P0**
- 未说明 API 的作用 → **P1**
- 未说明参数的来源 → **P1**

**不判定为问题：**
- 未展示所有 API（只要展示核心 API 即可）
- API 调用的详细业务逻辑

#### 4.5 生命周期管理（如适用）
**判定为问题：**
- SDK 需要手动释放资源但未说明 → **P1**

#### 4.6 多平台适配
**判定为问题：**
- 支持多平台但未分别说明集成流程 → **P0**

---

### Step 5：检查验证集成

**检查项：**
- 是否说明如何验证集成（运行项目、查看日志等）
- 是否说明集成成功的标志

**判定为问题：**
- 未说明如何验证集成 → **P0**
- 未说明成功的标志 → **P1**

---

### Step 6：检查快速体验功能（如适用）

**适用场景：** 文档定位为"快速接入"或"快速开始"

**检查项：**
- 是否说明要体验的核心功能
- 是否提供完整的代码示例
- 是否说明如何验证功能是否生效
- 如功能需要多端协作，是否说明如何验证

**判定为问题：**
- 文档定位为"快速接入/快速开始"，但未提供快速体验功能的步骤 → **P0**
- 未说明要体验的核心功能 → **P0**
- 未提供代码示例 → **P0**
- 功能需要多端协作但未说明 → **P1**

**不判定为问题：**
- 文档定位为"集成文档"而非"快速开始"，未提供快速体验功能

---

### Step 7：检查后续步骤引导（强制）

**检查项：**
- 是否提供后续文档的链接（功能指南、API 文档）

**判定为问题：**
- 完全缺少后续步骤引导 → **P1**
- 后续文档链接失效 → **P0**

---

### Step 8：检查常见问题（可选）

**判定为问题：**
- "常见问题"章节为空或只有标题 → **P1**
- 问题只有现象没有解决方案 → **P1**

**不判定为问题：**
- 集成非常简单，确实没有常见问题（文档可不包含此章节）

---

## Output Format（强制）

请输出 JSON 格式，如果没有问题则输出空数组 `[]`。

**⚠️ 重要说明**：此 JSON 输出用于在内存中汇总审查结果，**不单独保存为文件**。最终会生成汇总的 JSON 文件（包含所有审查模块结果）和 Markdown 报告。

```json
{
  "integration_issues": [
    {
      "type": "文档定位不清/前提条件缺失/安装步骤不明确/初始化配置缺失/验证方法缺失/快速体验缺失/后续引导缺失/常见问题缺失",
      "severity": "P0/P1/P2",
      "location": "问题所在章节",
      "original_description": "有问题的原文（如适用）",
      "description": "问题描述",
      "suggestion": "修改建议"
    }
  ]
}
```

**Severity 判定标准：**
- **P0**：会导致用户**无法完成集成**或**无法验证集成成功**
- **P1**：会导致用户**操作困难或需要额外摸索**
- **P2**：不影响集成，但**影响体验或理解**
