# 技术文档审查系统

一个基于分层架构的 AI 技术文档审查系统，用于自动审查技术文档（Demo、集成、功能指南、API 参考）的准确性、完整性和规范性。

## ✨ 主要功能

- **格式审查**：检查中英文空格、代码块格式、排版规范
- **代码审查**：验证代码示例的正确性和可运行性
- **语言审查**：检查语言表达的准确性和专业性
- **逻辑审查**：确保文档逻辑完整、步骤清晰
- **术语审查**：保证术语使用的一致性
- **业务审查**：针对不同文档类型（API、Demo、功能指南、集成文档）进行专项检查

## 🏗️ 架构设计

采用三层架构设计：

- **L0 全局层**：项目级规则和审查流程
- **L1 公共层**：通用审查标准（格式、代码、语言、逻辑、术语）
- **L2 业务层**：特定文档类型的审查规则

## 📂 项目结构

```
doc-review/
├── .cursorrules              # 全局规则和流程
├── CHANGELOG.md              # 规则变更历史
├── input-docs/               # 待审查文档目录
├── reports/                  # 审查报告输出目录
└── skills/                   # 审查技能库
    ├── _shared/              # 公共层审查标准
    └── review-*/             # 业务层审查规则
        ├── review-api/       # API 文档审查
        ├── review-demo/      # Demo 文档审查
        ├── review-feature/   # 功能指南审查
        └── review-integration/ # 集成文档审查
```

## 🚀 快速开始

1. 将待审查的文档放入 `input-docs/` 目录
2. 在 Cursor 中与 AI 对话，要求审查文档
3. 系统会自动执行审查流程，生成报告到 `reports/` 目录

## 📊 审查标准

- **P0（严重）**：必须修复 - 代码冲突、安全泄露、链接失效、核心信息缺失
- **P1（警告）**：建议修复 - 格式违规、表述模糊、逻辑断链
- **P2（建议）**：可选优化 - 语气优化、风格建议

## 🔍 审查流程

1. 读取文档内容
2. 执行共享层审查（格式、代码、语言、逻辑、术语）
3. 判断文档类型并执行业务层审查
4. 生成审查报告（JSON + Markdown）
5. 自动质量检查

## 📝 使用示例

```
请审查 input-docs/xxx.md 文档
```

系统会自动：
- 识别文档类型
- 执行相应的审查规则
- 生成详细的审查报告
- 提供修改建议

## ⚠️ 核心原则

- **零误报原则**：不确定的问题标记为 `[需人工确认]`，严禁误报
- **证据导向**：每个问题必须有确凿证据
- **可操作性**：每个问题都提供具体的修改建议
