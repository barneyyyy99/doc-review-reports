## 🔴 P0 问题（严重，必须修复）

### 1. [feature] 时序流程不完整（应用内分享缺少停止说明）

**位置**：应用内分享 章节

**原文**：
> 应用内分享的方案非常简单，只需要调用 TRTC SDK 提供的接口 TRTCCloud#startScreenCaptureInApp 并传入编码参数 TRTCVideoEncParam 即可。

**问题描述**：应用内分享章节仅说明了如何调用 `startScreenCaptureInApp` 开始屏幕分享，完全没有说明如何停止应用内屏幕分享（对应 `stopScreenCapture`），导致功能生命周期不完整，开发者无法得知如何结束分享。

**修改建议**：在应用内分享章节末尾补充停止说明，例如："调用 `TRTCCloud#stopScreenCapture` 可随时中止屏幕分享。"并补充对应代码示例。

---

### 2. [logic] 步骤遗漏（系统控制中心触发步骤缺少关键操作说明）

**位置**：步骤3：对接主 App 端的接收逻辑 → 第3条

**原文**：
> 等待用户触发屏幕分享。如果不实现步骤4中的"触发按钮"，屏幕分享就需要用户在 iOS 系统的控制中心，通过长按录屏按钮来触发……

**问题描述**：说明用户从系统控制中心触发录屏时，未说明用户需要在弹出列表中选择本 App 的扩展。对于初次接入的开发者而言，如果未在弹出列表中选择正确的扩展，屏幕分享将无法启动，这是一个容易让开发者卡住的操作步骤。

**修改建议**：补充说明："用户在控制中心长按录屏按钮后，需在弹出的列表中选择本 App 对应的录屏扩展，方可触发屏幕分享数据传输。"

### 4. [feature] 核心 API 参数缺少说明

**位置**：步骤3：对接主 App 端的接收逻辑 → 第2条

**原文**：
> 调用 TRTCCloud#startScreenCaptureByReplaykit:appGroup: 方法，并传入步骤1中设置的 AppGroup，让 SDK 进入"等待"状态。

**问题描述**：`startScreenCaptureByReplaykit:appGroup:` 是跨应用屏幕分享的核心启动 API，其第一个参数（`videoEncParam`，`TRTCVideoEncParam` 类型）在正文中未作任何说明（含义、必填/可选、传 nil 的行为）。代码示例中使用了该参数，但正文缺乏说明，不符合核心 API 参数必须在正文中说明的要求。

**修改建议**：在步骤3第2条正文中补充参数说明：
- `videoEncParam`（`TRTCVideoEncParam`，可选）：视频编码参数，传 `nil` 时 SDK 沿用屏幕分享前的编码参数；
- `appGroup`（`NSString`，必填）：App Group Identifier，需与步骤1中配置的值一致。

### 5. [feature] 回调处理缺失

**位置**：跨应用分享 → 步骤3：对接主 App 端的接收逻辑

**原文**：代码示例仅包含 `onScreenCaptureStarted`，正文未说明屏幕分享停止或异常时的回调处理。

**问题描述**：代码示例仅展示了 `onScreenCaptureStarted` 成功回调，正文未说明当屏幕分享因异常终止（如 Extension 进程被系统杀死）时，主 App 端应如何感知并响应。开发者无法依据文档处理分享中断场景，影响接入质量。

**修改建议**：在步骤3中补充说明停止/异常回调，例如："`onScreenCaptureStopped`：屏幕分享停止时触发（包含主动停止和异常停止），回调参数包含停止原因，建议在此回调中更新 UI 状态。"
