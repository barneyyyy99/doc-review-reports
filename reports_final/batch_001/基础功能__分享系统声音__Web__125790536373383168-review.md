**问题描述**：

### 4. [功能] 回调/事件处理缺失 — 未说明如何确认系统音频已生效

**位置**：实现流程 > 代码示例

**问题描述**：
文档仅说明远端用户会收到 `TRTC.EVENT.REMOTE_AUDIO_AVAILABLE` 事件，但未说明本地发布方如何确认系统音频是否成功被采集并发布。此外，用户在浏览器对话框中可能不勾选"Share system audio"，此时系统音频轨道不会被包含，但文档对该情况无任何说明，开发者无法判断是否需要处理这一分支。

**原文**：
> 在弹出的屏幕分享选择框中勾选 **Share system audio**，点击 **Share** 即可。在进房发布后，房间内其他用户会收到 TRTC.EVENT.REMOTE_AUDIO_AVAILABLE 事件。

**修改建议**：
补充以下说明：
1. `startScreenShare` 返回的 Promise resolve 时表示屏幕分享（含系统音频轨道，若用户勾选了选项）已成功发布。
2. 若用户在浏览器对话框中未勾选"Share system audio"，系统音频轨道不会被采集，建议说明如何检测（例如：检查 `startScreenShare` 返回的本地流中是否包含 audio track，或通过对比预期与实际 track 数量来判断）。
