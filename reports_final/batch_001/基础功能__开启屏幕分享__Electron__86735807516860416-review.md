## P0 问题（必须修复）

### LOGIC-001 · 前后描述矛盾

- **位置**: 步骤4：观看屏幕分享
- **原文**: "当房间里有一个用户启动了屏幕分享，会通过辅路进行分享。"
- **问题**: 文档在开头介绍了主路分享和辅路分享两种方案，但步骤4（观看屏幕分享）的正文和代码示例均只处理辅路场景（`onUserSubStreamAvailable` + `TRTCVideoStreamTypeSub`），完全未说明当发送方使用主路分享时，观看端如何订阅画面。开发者按照步骤4操作，将无法订阅主路屏幕共享流，导致主路分享场景无法完成。
- **建议**: 在步骤4中补充主路分享的观看说明：主路分享时监听 `onUserVideoAvailable` 事件，调用 `startRemoteView(userId, dom, TRTCVideoStreamTypeBig)` 订阅；辅路分享则如现有示例，监听 `onUserSubStreamAvailable` 并调用 `startRemoteView` with `TRTCVideoStreamTypeSub`。

## P1 问题（建议修复）

### LANG-001 · 视角混乱

- **位置**: 介绍段落辅路分享/主路分享列表项；步骤2；步骤3
- **原文**: "这是腾讯会议的使用方案，您可以在调用 `startScreenCapture` 接口时……"
- **问题**: 文档将"腾讯会议的使用方案"与开发者的实现行为混同，"您"的使用在部分场景下造成视角混乱。技术文档的读者是开发者，应保持开发者视角，不宜将最终产品使用场景（腾讯会议）与开发者操作混同表述。
- **建议**: 将操作引导句改为开发者视角的直接描述。例如：'可在调用 `startScreenCapture` 接口时，通过将 `TRTCVideoStreamType` 参数指定为 `TRTCVideoStreamTypeSub` 来启用该模式（腾讯会议采用此方案）。' 全文中有多处，请批量修改。

### LOGIC-002 · 关键分支场景缺失

- **位置**: 介绍段落（主路分享 vs 辅路分享）
- **原文**: "Electron 平台下的屏幕分享支持主路分享和辅路分享两种方案"
- **问题**: 文档列出了两种方案，但未说明开发者如何选择，两种方案的适用场景不明确。全文操作步骤仅以辅路分享为示例，开发者不清楚主路分享的适用条件。
- **建议**: 在介绍两种方案时，补充选择建议，例如：'如需同时上行摄像头画面，建议使用辅路分享；如只需上行屏幕画面（摄像头通道互斥），可使用主路分享。'

---

### CODE-001 · 代码选择器无效

- **位置**: 步骤2：开始屏幕分享 代码块
- **原文**: `const screenshareDom = document.querySelector('screen-dom');`
- **问题**: `'screen-dom'` 是无效的 CSS 选择器（缺少 `.` 类选择器或 `#` ID 选择器前缀），`querySelector` 会始终返回 `null`。将 `null` 传入 `startScreenCapture` 的第一个参数可能导致运行时异常。对比步骤4中的 `document.querySelector('.remote-user')` 写法正确，形成不一致。
- **建议**: 修改为有效选择器，例如：`const screenshareDom = document.querySelector('#screen-dom');` 或 `document.querySelector('.screen-dom')`，并与实际 DOM 结构保持一致。

---

### FEAT-002 · 依赖关系缺失（前置条件未说明）

- **位置**: 文档开头 / 操作步骤前
- **原文**: （文档直接从获取分享目标开始，无前置条件说明）
- **问题**: 屏幕分享功能依赖已完成 SDK 初始化并加入 TRTC 房间，但文档未说明此前置条件，也未提供相关文档链接。开发者若未进入房间直接调用屏幕分享接口，将无法正常工作。
- **建议**: 在文档开头增加前置条件说明，例如：'前置条件：需先完成 SDK 初始化并加入 TRTC 音视频房间（详见 [快速集成文档](链接)）。'

---

### FEAT-003 · 核心 API 参数缺少正文说明

- **位置**: 步骤2：开始屏幕分享
- **原文**: "使用 `startScreenCapture` 接口可以启动屏幕分享。"
- **问题**: `startScreenCapture` 是本文档的核心 API，但其三个参数中，`view`（DOM 元素，用于本地预览）和 `streamType`（`TRTCVideoStreamType`，选择主路或辅路）的含义未在正文中说明，仅能从代码示例中推断。按照功能指南规范，核心 API 参数需在正文中说明，不能仅依赖代码注释。
- **建议**: 在步骤2正文中补充参数说明：1) `view`：用于本地预览的 DOM 元素，不需要预览时可传 `null`；2) `streamType`：`TRTCVideoStreamTypeSub` 为辅路，`TRTCVideoStreamTypeBig` 为主路；3) `encParam`：编码参数，详见步骤3。
